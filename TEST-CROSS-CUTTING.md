# Cross-Cutting Test Report — Polymath (non-domain surfaces)

## Summary

Walked every non-domain surface and global chrome. The big-ticket bugs cluster in: (1) the Home hero/search interaction (typing filters nothing until the catalogue is manually revealed), (2) SW precache referencing missing icon files, (3) UTC-vs-local timezone mismatch in three places that key off `xpByDay`, (4) Search lacks the 3-char floor the spec calls for and has a stale "Esc" hint with no handler, (5) My-list bookmarks deep-link only to the domain root (not the bookmarked concept), and (6) the TopBar primary CTA points at a non-existent `#domains` anchor for first-time users. The ConceptOfTheDay deep-link fix is correctly in place. Onboarding, focus traps, FSRS shortcuts, Toaster, RouteAnnouncer, Coachmarks, sitemap and noindex are correct. Several quality issues (duplicate mastery formulas, hardcoded SW shell, stale Privacy claim on About) are worth a sweep.

## A11y findings

- **MegaMenu / PracticeMenu**: ARIA wiring correct (`aria-haspopup`, `aria-expanded`, `aria-controls`). Focus trap on MegaMenu present; focus restore happens unconditionally on cleanup which can interfere with link navigation (P2 above).
- **CommandPalette**: combobox + listbox + `aria-activedescendant` correctly wired. Good.
- **Toaster**: `role=status aria-live=polite aria-atomic=false`. Correct. (Layout overlap with BottomNav is the only issue — P1 above.)
- **RouteAnnouncer**: single instance in `app/layout.tsx`, polite live region, reads `<main h1>` after 80ms tick. Good.
- **Onboarding**: focus trap present, no Escape close (P1 above), focus-restore handled by `useFocusTrap`.
- **SaveLoadModal** (in TopBar.tsx): focus trap + Escape close + click-outside close. Good.
- **Coachmarks**: `role=dialog` but only the X button can close it. No Escape handler. Add `keydown` Esc → finish.
- **TodayCard non-clickable Stats** (XP, Streak, Daily quest): not focusable, not announced as interactive. Acceptable if they're informational; flag if "Daily quest: Set up" is meant to be a CTA.
- **`SearchView` input** has stale `<kbd>esc</kbd>` chip with no behaviour — misleading for keyboard users.
- **BottomNav** items lack `aria-label` describing the due count (`<span>{due}</span>` is read raw). Wrap due badge with `aria-label={`${due} cards due`}` or hide it via `aria-hidden` and add an sr-only span.
- **`<a href="#all-domains">`** in StartHereBento / MoreForYouRow does not reveal the catalogue (UX bug, doubles as a11y bug for screen readers — anchor lands on an empty `<div>`).
- **`html lang="en"`** present. Good. `suppressHydrationWarning` used since theme is applied via JS post-mount.

## Performance findings

- **Dashboard** loads all 15 domain payloads in parallel on mount (line 22-30). On a slow connection this is 15 simultaneous fetches before the radar renders. Consider sequencing or only loading started domains for the radar, and lazy-loading the rest only when capability strings are computed.
- **ReviewSession** does the same (line 22-32) but at least filters to `s.startedDomains` when any exist.
- **SearchView** fan-out (line 68-86) loads all 15 domains the first time a non-empty query is typed. Fine, but no progress UI — user sees "No matches yet" until each domain finishes.
- **ConnectionsGraph**, **Dashboard**, **ReviewSession**, **SearchView** each independently call `loadDomain(id)` for the same domains. No shared cache between them. Module-level `Map<string, Promise<Domain>>` in `data/domains.ts` would avoid duplicate fetches across surfaces in a single session.
- **StreakHeatmap** renders 365 SVG `<span>` cells with inline title/aria-label on every parent re-render. Memoized array good; consider `React.memo` on the component itself to short-circuit re-renders when `xpByDay` reference hasn't changed.
- **ServiceWorker** install will currently fail (missing icons → `addAll` rejects atomically). When fixed, network-first navigation strategy will re-fetch HTML on every page load — fine for freshness, but no `stale-while-revalidate` means a user with patchy network sees a spinner before fallback.
- **Toaster `setToasts(...slice(-3))`**: capped at 3 visible — good. But `setTimeout` cleanup is per-batch; many fast XP bumps could leak orphan timers. Acceptable in practice.

## SEO findings

- **Sitemap** (`app/sitemap.ts`): correctly excludes `/review`, `/dashboard`, `/skill-map`, `/my-list`, `/settings`, `/share`. Includes `/`, `/about`, every domain root, every tab segment, every subdomain, every concept deep page. Looks complete.
- **Per-page `robots: { index: false, follow: false }`**: present on review/dashboard/my-list/settings/share. SkillMap is `index: false, follow: true` (so SE engines can still discover linked content) — defensible. Good.
- **robots.txt**: only disallows `/api/` and `/_next/`. Add explicit `disallow` for the noindex'd surfaces as belt-and-braces (P1 above).
- **JSON-LD `WebApplication`** in `app/layout.tsx` is valid; `metadataBase`, canonical alternates, OpenGraph, Twitter cards all wired. Good.
- **RSS** (`app/domains.rss/route.ts`): 15 channel items, `force-static`, cache-control headers (`max-age=600, s-maxage=86400`). `<link rel="alternate" type="application/rss+xml">` present in layout. Good. Minor: feed lacks a `pubDate` per item — most readers will accept this, but adding the build date helps.
- **`<head>` icon refs** point to `/icon-192.png` and `/apple-touch-icon` (same file) which don't exist (see P0). Browsers will show a 404 in devtools and fall back to default favicon.
- **`canonical: "/"`** is set globally in layout metadata; per-page metadata correctly sets its own canonical for review/dashboard/my-list/settings/skill-map/share. Good.
- **Featured-today rotation** keyed off UTC (HubCatalogue.tsx:14) means SE crawlers from different regions can see different featured domain at the day boundary. Cosmetic.
